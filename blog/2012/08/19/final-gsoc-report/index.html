Final GSoC Report

Aug 19th, 2012

Wait, august 19th? Where’s my summer? I feel like i just started this GSoC!

Alas, it’s already coming to an end. Those two month have been amazing. Let’s head back and summarize what I’ve been doing during this time.

1. The Proposal

When I felt I was going to pass my exams and thus was going to have an exams-free summer, I knew I had to seize the opportunity to try to be selected for a GSoC. I talked about it with Guillaume Desmottes and we got to the idea of my proposal.

I was really happy to be selected and looking forward to the end of my exams to start working.

2. The Work’s Beginning

Around mid june it was finally time to get started. As I stated in my first weekly report, I begun by fixing some bugs to get in touch with Empathy’s code base. I then ported empathy-nautilus-sendto to the RosterView instead of EmpathyContactChooser.

It’s worth noting that Guillaume’s work on the RosterView between my proposal and my work’s beginning quite modified the initial planning.

3. The Model

The next step of my work was one of the most important : I created empathy-roster-model and empathy-roster-model-manager and moved all the logic out of the view to the model. This took longer than what Guillaume and I thought initially, but it got merged into master around the beginning of august.

During the model’s development I created a new test app in which I created 2 views using the same model. This helped me understanding what goal Guillaume wanted me to achieve here.

4. The ModelAggregator

empathy-roster-model-manager worked great for Empathy but it was way too Empathy specific to be usable elsewhere. Since the ultimate goal of the project is to create a library that any project could use I created empathy-roster-model-aggregator, a new implementation of empathy-roster-model relying on FolksIndividualAggregator.

5. A New Internal Lib

Once empathy-roster-model-aggregator was done and working as it should, I moved the view, the model (along with model-aggregator, but not model-manager) and various needed tools like EmpathyRosterContact or EmpathyRosterGroup to a new directory inside Empathy’s tree.

This is one step closer to the goal of the external library.

6. Gtk-doc and GIR

If we want this library to be widely used along GNOME projects, it had to be documented. I therefore added gtk-doc to the project and wrote some doc.

Using Gobject-introspection was also one step in the roadmap. I discovered how gir functions, created EmpathyRoster-0.1.gir and wrote a small example test app in Python.

7. The Future

So, what’s left to do?

  • A part of my code has yet to be reviewed, Guillaume being on holidays.
  • The library should be moved out of Empathy’s tree.
  • The gir annotations should be completed to eliminate warnings.
  • There’s probably more doc to write.

8. The Asides

In addition to what I’ve already talked about I also worked on other things. One of the most tiring ones was idling on a lot of new IRC channels. I also fixed a few bugs in Empathy and worked on a patch for Folks.

Unlike what I feared, I found myself quite enjoying writing my Weekly Reports. It was a nice way to have to take time to evaluate the week’s productivity.

9. The Results

To sum things up, let’s say that :

  • My coding skills have dramatically improved
  • I finally contributed to Open Source Software
  • I’ve had the best conceivable summer job. Not once did I start to work feeling I “had to” (excepted maybe for the doc writing step, let’s be honest).
  • I’ve acquired more knowledge about the various aspects of software development (at least regarding GNOME softwares).
  • I’ve lost time on really stupid errors.
  • I’m quite satisfied of my work.
  • I have more IRC chans to idle on.
  • I plan to continue coding and contributing.

TL;DR: Amazing experience. 10/10, would do again.
